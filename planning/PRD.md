# 기획 파트 — 상세 PRD

## 1. 기능 명세

### 1-1. 프로젝트 갤러리 (메인 화면)

| 항목 | 내용 |
|------|------|
| 목적 | 워크숍 참여자들의 프로젝트를 한눈에 탐색 |
| 형태 | 카드형 그리드 (반응형 2~4열) |
| 카드 구성 요소 | 썸네일 이미지, 프로젝트 제목, 참여자 이름, 한 줄 소개, 영감 수(좋아요 카운트) |
| 정렬 | 최신순(기본), 영감 많은 순 |
| 필터/검색 | 워크숍 기수별 필터, 키워드 검색 |

**동작 정의:**
- 페이지 진입 시 전체 프로젝트가 카드 그리드로 노출된다
- 카드 호버 시 썸네일에 미세한 스케일업 효과
- 카드 클릭 시 프로젝트 상세 페이지로 이동
- 스크롤 시 추가 프로젝트 로드 (무한 스크롤 또는 페이지네이션 — MVP에서는 전체 로드)

### 1-2. 프로젝트 상세 페이지

| 항목 | 내용 |
|------|------|
| 목적 | 개별 프로젝트의 자세한 정보 확인 |
| 구성 요소 | 프로젝트 제목, 참여자 이름/프로필, 대표 이미지, 프로젝트 설명(마크다운), 사용 도구/기술 태그, 프로젝트 링크(외부), 영감 버튼 |

**동작 정의:**
- 갤러리에서 카드 클릭 시 상세 페이지 진입
- 프로젝트 설명은 마크다운 렌더링 지원
- 외부 링크 클릭 시 새 탭에서 열림
- 하단에 "다른 프로젝트 둘러보기" 영역으로 갤러리 복귀 유도

### 1-3. 영감 반응 (좋아요)

| 항목 | 내용 |
|------|------|
| 목적 | 참여자 간 영감 교환, 동기 부여 |
| 형태 | 하트/스파클 아이콘 버튼 + 카운트 |
| 인증 | 비로그인 사용 가능 (브라우저 기반 중복 방지) |

**동작 정의:**
- 갤러리 카드 및 상세 페이지에서 영감 버튼 클릭 가능
- 클릭 시 카운트 +1, 아이콘 애니메이션 재생
- 동일 브라우저에서 같은 프로젝트에 중복 반응 방지 (localStorage)
- 재클릭 시 반응 취소 (토글 방식)

### 1-4. 프로젝트 등록

| 항목 | 내용 |
|------|------|
| 목적 | 워크숍 참여자가 자신의 프로젝트를 아카이브에 추가 |
| 입력 필드 | 프로젝트 제목(필수), 한 줄 소개(필수), 썸네일 이미지(필수), 프로젝트 설명(선택), 참여자 이름(필수), 프로젝트 링크(선택), 사용 도구/기술 태그(선택), 워크숍 기수(필수) |
| 인증 | MVP에서는 관리자 검수 후 게시 (폼 제출 → 관리자 승인) |

**동작 정의:**
- 등록 폼 페이지에서 정보 입력
- 이미지는 파일 업로드 또는 URL 입력
- 제출 후 "등록 요청 완료" 안내 → 관리자 승인 후 갤러리에 노출
- 입력값 유효성 검사 (필수 필드 미입력 시 안내 메시지)

## 2. 사용자 시나리오

### 시나리오 A: 워크숍 참여자가 프로젝트를 등록한다

| 항목 | 내용 |
|------|------|
| **Who** | 셀피쉬클럽 워크숍 수료자 (예: AI 워크숍 3기 참여자) |
| **When** | 워크숍 종료 후, 자신의 결과물을 정리한 시점 |
| **Where** | Workshop Archive 사이트의 등록 페이지 |
| **Why** | 자신의 프로젝트를 기록하고, 다른 참여자에게 공유하여 영감을 주고 싶어서 |
| **How** | 1. 메인 페이지에서 "프로젝트 등록" 버튼 클릭 → 2. 폼에 프로젝트 정보 입력 → 3. 썸네일 이미지 업로드 → 4. 제출 → 5. 관리자 승인 후 갤러리에 노출 |

### 시나리오 B: 외부 방문자가 프로젝트를 탐색한다

| 항목 | 내용 |
|------|------|
| **Who** | 셀피쉬클럽에 관심 있는 잠재 참여자 |
| **When** | SNS나 지인 추천으로 사이트를 알게 된 시점 |
| **Where** | Workshop Archive 메인 갤러리 |
| **Why** | 워크숍에서 어떤 프로젝트를 만드는지 궁금해서 |
| **How** | 1. 사이트 접속 → 2. 갤러리에서 카드 스크롤하며 탐색 → 3. 흥미로운 프로젝트 카드 클릭 → 4. 상세 내용 확인 → 5. 영감 버튼 클릭 → 6. 다른 프로젝트도 둘러봄 |

### 시나리오 C: 참여자가 다른 프로젝트에 영감을 받는다

| 항목 | 내용 |
|------|------|
| **Who** | 이미 프로젝트를 등록한 워크숍 참여자 |
| **When** | 커뮤니티 채널에서 아카이브 링크를 보고 접속 |
| **Where** | 갤러리 → 상세 페이지 |
| **Why** | 같은 기수 동기들의 프로젝트가 궁금하고, 영감을 주고 싶어서 |
| **How** | 1. 갤러리에서 기수 필터 적용 → 2. 동기의 프로젝트 확인 → 3. 영감 버튼으로 반응 → 4. 여러 프로젝트에 영감 남김 |

## 3. MVP 범위

### 필수 (MVP — v1.0)

| 기능 | 설명 |
|------|------|
| 프로젝트 갤러리 | 카드형 그리드, 최신순 정렬 |
| 프로젝트 상세 페이지 | 제목, 설명, 이미지, 태그, 링크 |
| 영감 반응 | 비로그인 좋아요 (localStorage 기반 중복 방지) |
| 프로젝트 등록 폼 | 기본 정보 입력 + 관리자 승인 방식 |
| 반응형 레이아웃 | 모바일/태블릿/데스크탑 대응 |

### 추후 (v1.1+)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 워크숍 기수별 필터 | 기수 선택하여 해당 프로젝트만 보기 | 높음 |
| 키워드 검색 | 제목/설명/태그 기반 검색 | 높음 |
| 영감 많은 순 정렬 | 인기 프로젝트 정렬 옵션 | 중간 |
| 참여자 프로필 | 참여자별 프로젝트 모아보기 | 중간 |
| 소셜 공유 | 프로젝트 링크를 SNS에 공유 (OG 태그) | 중간 |
| 코멘트 기능 | 프로젝트에 텍스트 피드백 남기기 | 낮음 |
| 관리자 대시보드 | 등록 요청 관리, 프로젝트 편집/삭제 | 낮음 |

## 4. 우선순위 매트릭스

영향력(Impact)과 구현 난이도(Effort)를 기준으로 분류:

### 높은 영향 + 낮은 난이도 (즉시 실행)
- 프로젝트 갤러리 (카드 그리드)
- 프로젝트 상세 페이지
- 영감 반응 (좋아요)

### 높은 영향 + 중간 난이도 (MVP 포함)
- 프로젝트 등록 폼
- 반응형 레이아웃

### 중간 영향 + 낮은 난이도 (v1.1 우선)
- 기수별 필터
- 영감순 정렬
- OG 태그 (소셜 공유)

### 중간 영향 + 높은 난이도 (추후)
- 키워드 검색
- 참여자 프로필 페이지
- 코멘트 기능
- 관리자 대시보드

## 5. 데이터 구조 (참고)

프로젝트 등록 시 필요한 데이터 모델:

```
Project {
  id: string (자동 생성)
  title: string (프로젝트 제목)
  subtitle: string (한 줄 소개)
  description: string (상세 설명, 마크다운)
  thumbnail: string (썸네일 이미지 URL)
  authorName: string (참여자 이름)
  projectUrl: string (외부 링크, 선택)
  tags: string[] (사용 도구/기술)
  cohort: string (워크숍 기수)
  inspirationCount: number (영감 카운트)
  status: 'pending' | 'approved' (승인 상태)
  createdAt: datetime (등록 일시)
}
```
